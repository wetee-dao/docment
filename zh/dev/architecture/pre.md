# 代理重加密 (PRE)

代理重加密 (PRE) 是 WeTEE 的核心隐私特性，它允许在不向平台或工作节点泄露原始数据的情况下，实现安全的数据共享和访问控制。

## 工作原理

1.  **用户加密:** 用户使用自己的密钥加密敏感数据（例如机密任务的输入数据）。
2.  **重加密密钥:** 用户生成一个针对特定 TEE 工作负载公钥的特殊“重加密密钥”。
3.  **代理 (侧链):** WeTEE 侧链（代理）使用重加密密钥将用户的加密数据转换为只能由获得授权的 TEE 工作负载解密的新版本。
4.  **TEE 内解密:** TEE 工作负载在其安全硬件飞地内解密数据，进行处理并输出结果。

## 优势

*   **零泄露:** 在重加密过程中，侧链节点和 Worker 管理员都无法看到用户的数据。
*   **细粒度访问控制:** 用户可以通过销毁重加密密钥或在侧链上更改策略来撤销访问权限。
*   **可审计性:** 每一个重加密事件都会记录在侧链上以供审计。
