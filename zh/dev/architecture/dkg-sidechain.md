# DKG 与侧链

WeTEE 利用第二层侧链来处理敏感操作和去中心化密钥管理，确保没有任何单一实体能够完全控制系统的根秘密。

## 分布式密钥生成 (DKG)

DKG 是一种协议，由一组节点集体生成公钥/私钥对。
*   **无单一信任源:** 私钥永远不会在任何单个节点上完整地重建。相反，它以“份额”的形式存储在所有参与节点中。
*   **阈值密码学:** 使用私钥进行的操作（如签名或解密）需要达到阈值数量的节点（例如 2/3）使用各自的份额参与。
*   **安全性:** 这可以防止节点被攻破或恶意管理员的操作，因为他们只能获得密钥的一个无用片段。

## 侧链的作用

基于 CometBFT 构建的侧链充当 DKG 的协调层：
1.  **节点选择:** 选举授权参与 DKG 的工作节点集。
2.  **周期管理:** 通过“周期 (Epochs)”处理密钥轮换和节点成员变更。
3.  **治理同步:** 同步主链中影响侧链运行的治理决策。
