# 从 Docker 到 TEE

WeTEE 允许开发者将标准的 Linux 容器通过极少的修改即可运行在受信任执行环境 (TEE) 中。这是通过 EGO 等 LibOS 技术实现的。

## 开发工作流

1.  **准备 Docker 镜像:** 确保您的应用程序可以作为标准的 Docker 容器正常运行。
2.  **EGO 打包:** 使用 EGO 工具集为 SGX 飞地签名并打包您的二进制文件。这通常涉及定义一个 `enclave.json` 配置文件。
3.  **上传至 WeTEE:** 通过 WeTEE dApp 或 CLI，指定您的 Docker 镜像和必要的 TEE 配置。
4.  **远程度量证明:** 当容器启动时，WeTEE 的 Worker 会生成 DCAP 远程度量报告，向网络证明代码正运行在真实的 SGX 飞地内。
5.  **执行:** 您的应用程序在具有完整硬件隔离和加密的环境中运行。

## 最佳实践

*   **默认无状态:** 尽可能将您的 TEE 容器视为无状态的，或者使用 WeTEE 的机密存储来实现持久化。
*   **精简基础镜像:** 使用小型基础镜像（如 Alpine）以减小可信计算基 (TCB)。
*   **安全环境变量:** 使用 WeTEE 的密钥管理系统将敏感环境变量直接注入到 TEE 中。
