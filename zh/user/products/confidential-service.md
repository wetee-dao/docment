# 部署机密应用 (Confidential Service)

**机密应用**（Service）适用于需要长时间在线运行的程序，比如 Web 网站、数据库或 API 服务。

### 为什么它很特殊？
当你部署一个机密应用时，它运行在一个完全加密的“硬件黑盒”中。即使是提供服务器的矿工，也无法看到你的代码逻辑或内存中的敏感数据。

### 如何部署？(只需 3 步)

#### 第一步：准备镜像
PulsWeb2 支持标准的 Docker 镜像。你只需要将你的镜像上传到公共仓库（如 Docker Hub）或私有仓库。

#### 第二步：填写配置
1.  登录 PulsWeb2 dApp。
2.  进入 **“Confidential Service”** 页面，点击 **“Create”**。
3.  **填写信息**：
    *   **Image**: 输入你的 Docker 镜像地址（例如 `nginx:latest`）。
    *   **Resources**: 选择你需要的 CPU 和内存大小。
    *   **Port**: 填写程序对外提供的端口。

#### 第三步：一键启动
点击 **“Deploy”** 按钮并签名。系统会自动在全网匹配合适的机密算力节点，通常在 10-30 秒内，你的应用就会上线。

### 监控与管理
*   **实时日志**：你可以在控制台直接查看程序的运行输出。
*   **状态验证**：点击“Verify”按钮，系统会向你展示该节点真实的硬件加密证明，确保你的隐私万无一失。
*   **弹性伸缩**：你可以随时增加或减少资源，或者停止服务以停止计费。

---

**场景建议**：如果你想运行一个私人的密码管理器、加密数据库，或者是需要处理用户隐私数据的后端服务，请选择“机密应用”。
