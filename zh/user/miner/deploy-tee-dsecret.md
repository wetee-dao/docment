# 部署可信子网 `tee-dsecret`（安全共识节点）

`tee-dsecret` 是 PulsWeb2 的**可信子网（Trusted Subnet）**组件，用于承载需要共识保障的关键安全流程（例如 DKG、成员管理、密钥轮换等）。

它的定位与 Worker 不同：
- `tee-worker`：负责跑用户的机密计算（算力节点）
- `tee-dsecret`：负责提供安全共识与子网能力（安全共识节点）

## 你需要准备什么

由于不同环境（内测/私有部署/合作网络）对参数与拓扑有差异，这里只给出**不依赖“发币/代币经济学”的通用准备项**：

- 一台或多台长期稳定运行的服务器（建议至少 3 节点用于容错）
- 固定公网 IP 或稳定的内网互通网络
- 持久化存储（用于保存共识与状态数据）
- 运维能力：监控、日志、备份与升级窗口

## 部署方式（建议）

当前工作空间里 Worker 侧是以 Kubernetes/K3s 方式部署的（可参考 Worker 安装文档）。`tee-dsecret` 也建议采用同样的方式统一运维：

1. 准备 Kubernetes/K3s 集群
2. 为 `tee-dsecret` 预留存储卷（PVC）与网络端口
3. 按环境提供的配置文件启动（genesis/peers/validator 等）
4. 确认节点加入集群并开始共识

> 说明：`tee-dsecret` 的具体启动参数、端口、以及节点加入流程，取决于你所在网络的“子网配置包”。该配置包通常由网络运营方或管理员提供。

## 运行检查

部署完成后，建议至少确认：
- 节点之间能互相发现并建立连接（peers 正常）
- 共识高度持续增长（没有卡高度）
- 日志中没有持续报错（尤其是存储、网络、签名相关）
- 配置了监控与告警（CPU/内存/磁盘/网络）
